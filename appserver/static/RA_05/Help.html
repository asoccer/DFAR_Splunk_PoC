Control Help

This application was created as a framework. Each search utilizes eventtypes and tagging in order to more easily customize the integration of other data sources. This help section outlines the definition of the searches, and supplies a listing of the fields which may be used for reporting. Please take a look at the About page for more information on customization.

This control utilizes data from the `vulnerability_scan` search macro.

Vulnerabilities

The `vulnerability_scan` macro is defined as:

tag=vulnerability tag=report $filter$ `vulnerability_scan_filter`
   | fillnull value=unknown dvc_type, dest, dest_ip, dest_port_proto, os, severity, signature

In the event that there are filter-based modifications to the search that you'd like to employ permanently, you may edit the `vulnerability_scan_filter` to affect the results. Further adjustments may require modifying the `vulnerability_scan` macro.

Recommended Fields: | fillnull value=unknown dvc_type, dest, dest_ip, dest_port_proto, os, severity, signature

  • dest = the scanned host
  • dest_ip = the scanned hosts's IP
  • dest_port = the host's vulnerable port number
  • dest_port_proto = the host's vulnerable port protocol
  • os = the operating system signature of the host
  • severity = the severity of the vulnerability
  • signature = a simple description of the vulnerability
