Control Help

This application was created as a framework. Each search utilizes eventtypes and tagging in order to more easily customize the integration of other data sources. This help section outlines the definition of the searches, and supplies a listing of the fields which may be used for reporting. Please take a look at the About page for more information on customization.

This control utilizes data from the `account_management` , `logins` , `account_lockout` , and `terminated_user_activity` search macros. Additionally, lookups/account_mapping.csv is utilized as a source for account_type and organization fields, based on the user.

Account Management

The `account_management` macro is defined as:

tag=account (tag=management OR tag=lock OR tag=unlock) $filter$ `account_management_filter`
   | lookup account_mapping user
   | fillnull value=unknown signature, src, src_nt_domain, src_user, dest, dest_nt_domain, user

In the event that there are filter-based modifications to the search that you'd like to employ permanently, you may edit the `account_management_filter` to affect the results. Further adjustments may require modifying the `account_management` macro itself.

Recommended Fields:

  • user = the name of the affected user
  • src_user = the name of the user making the change
  • src = the source host, if present
  • dest = the host on which the event took place
  • signature = a simple description of the change

Account Lockouts

The `account_lockout` macro is defined as:

tag=account tag=lock $filter$ `account_lockout_filter`

In the event that there are filter-based modifications to the search that you'd like to employ permanently, you may edit the `account_lockout_filter` to affect the results. Further adjustments may require modifying the `account_lockout` macro itself.

Recommended Fields:

  • user = the name of the locked out user
  • dest = the host on which the account lockout took place
  • signature = a simple description of the event

Authentication

The `logins` macro is defined as:

tag=authentication user=* $filter$ `login_filter`
   | lookup account_mapping user
   | fillnull value=unknown user, app, src, src_ip, dest, action, account_type, organization

In the event that there are filter-based modifications to the search that you'd like to employ permanently, you may edit the `logins_filter` to affect the results. Further adjustments may require modifying the `logins` macro itself.

Recommended Fields:

  • user = the user logging in
  • app = the login app (i.e. su, win:remote, etc)
  • src = source host of the login
  • src_ip = source IP address of the login
  • dest = destination host of the login
  • action = this should state the success status (success or failure) of the event

Terminated Users

The `terminated_user_activity` macro is defined as:

tag::user=terminated

Recommended Fields:

  • user = the user logging in
  • src = source host of the login
  • dest = destination host of the login
  • action = this should state the success status (success or failure) of the event

Lookups

Some of the information in this report is provided by a lookup in order to add account_type and organization fields. The account_type should be one of: system, guest, temporary, application, group, or individual. By default, the account_type for a user will be set to "individual", if nothing is specified. The organization field should be populated with the user's parent organization at the desired granularity.
