Control Help

This application was created as a framework. Each search utilizes eventtypes and tagging in order to more easily customize the integration of other data sources. This help section outlines the definition of the searches, and supplies a listing of the fields which may be used for reporting. Please take a look at the About page for more information on customization.

Data for this control may originate from a scanning tool (e.g. nessus) or from a scripted input on the destination host (e.g. Splunk_TA_windows/bin/win_listening_ports.bat).

This control utilizes data from the `listening_ports` search macro.

Listening Ports

The `listening_ports` macro is defined as:

tag=port tag=listening tag=report $filter$ `listening_ports_filter`
   | fillnull value=unknown dvc_type, os, dest_port, dest_port_proto, dest_port_state, transport, app, app_version

In the event that there are filter-based modifications to the search that you'd like to employ permanently, you may edit the `listening_ports_filter` to affect the results. Further adjustments may require modifying the `listening_ports` macro itself.

Recommended Fields:

  • dest = host on which the port was detected
  • dvc_type = the device category (switch, server, etc)
  • dest_port = port number detected
  • dest_port_proto = port protocol detected
  • dest_port_state = the port state (open, closed, etc)
  • os = the operating system of the scanned host
  • transport = the port transport (tcp, udp)
  • app = the application bound to the scanned port
  • app_version = the application version bound to the scanned port
