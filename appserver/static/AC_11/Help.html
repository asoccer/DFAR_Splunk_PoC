Control Help

This application was created as a framework. Each search utilizes eventtypes and tagging in order to more easily customize the integration of other data sources. This help section outlines the definition of the searches, and supplies a listing of the fields which may be used for reporting. Please take a look at the About page for more information on customization.

This control utilizes data from the `session_lock` and `session_unlock` search macros. Additionally, lookups/account_mapping.csv is utilized as a source for account_type and organization fields, based on the user.

Session Locks

The `session_lock` macro is defined as:

tag=authentication tag=lock $filter$ `session_lock_filter` | lookup account_mapping user

In the event that there are filter-based modifications to the search that you'd like to employ permanently, you may edit the `session_lock_filter` to affect the results. Further adjustments may require modifying the `session_lock` macro itself.

Recommended Fields:

  • user = the name of the affected user
  • dest = the host on which the lock took place
