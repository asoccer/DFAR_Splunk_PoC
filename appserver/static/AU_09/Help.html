Control Help

This application was created as a framework. Each search utilizes eventtypes and tagging in order to more easily customize the integration of other data sources. This help section outlines the definition of the searches, and supplies a listing of the fields which may be used for reporting. Please take a look at the About page for more information on customization.

This control utilizes data from the `audit_cleared` and `audit_stopped` search macros.

Audit System Stoppage and Clearing Events

The `audit_cleared` macro is defined as:

tag=audit tag=cleared $filter$ `audit_cleared_filter`

The `audit_stopped` macro is defined as:

tag=audit tag=stopped $filter$ `audit_stopped_filter`

The 2 macros defined above are executed as a single search: (`audit_cleared`) OR (`audit_stopped`)

In the event that there are filter-based modifications to the search that you'd like to employ permanently, you may edit the `audit_cleared_filter` or the `audit_stopped_filter` to affect the results. Further adjustments may require modifying the `audit_cleared` or `audit_stopped` macros.

Recommended Fields:

  • dest = host where the network event occurred
  • action = this should state the success status (success or failure) of the event
  • signature = a simple description of the event
