Control Help

This application was created as a framework. Each search utilizes eventtypes and tagging in order to more easily customize the integration of other data sources. This help section outlines the definition of the searches, and supplies a listing of the fields which may be used for reporting. Please take a look at the About page for more information on customization.

This control utilizes data from the `audit_failures` search macro.

Audit Failures

The `audit_failures` macro is defined as:

tag=audit action=failure $filter$ `audit_failures_filter`

In the event that there are filter-based modifications to the search that you'd like to employ permanently, you may edit the `audit_failures_filter` to affect the results. Further adjustments may require modifying the `audit_failures` macro itself.

Recommended Fields:

  • dest = the host on which the event took place
