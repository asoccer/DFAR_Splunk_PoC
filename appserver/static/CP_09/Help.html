Control Help

This application was created as a framework. Each search utilizes eventtypes and tagging in order to more easily customize the integration of other data sources. This help section outlines the definition of the searches, and supplies a listing of the fields which may be used for reporting. Please take a look at the About page for more information on customization.

This control utilizes data from the `backup_event` search macro. Additionally, lookups/account_mapping.csv is utilized as a source for account_type and organization fields, based on the user.

Backup Events

The `backup_event` macro is defined as:

tag=backup $filter$ `backup_event_filter`
   | fillnull value=unknown dest, action, user, signature, vendor, account_type, organization

In the event that there are filter-based modifications to the search that you'd like to employ permanently, you may edit the `backup_event_filter` to affect the results. Further adjustments may require modifying the `backup_event` macro itself.

Recommended Fields:

  • user = the user running the backup
  • dest = destination host of the backup
  • action = the status of the backup event (success or failure)
  • signature = a simple description
  • vendor = the vendor of the backup product

Lookups

Some of the information in this report is provided by a lookup in order to add account_type and organization fields. The account_type should be one of: system, guest, temporary, application, group, or individual. By default, the account_type for a user will be set to "individual", if nothing is specified. The organization field should be populated with the user's parent organization at the desired granularity.
